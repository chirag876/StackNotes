01

Lists (Array Equivalent)

Slicing & indexing: arr[i], arr[-1], arr[start:end:step] — negative indices count from end. Slicing creates a new list.
List comprehension: [expr for x in iterable if condition] — cleaner and faster than manual loops in most cases.
sort() vs sorted(): sort() modifies in-place, returns None. sorted() returns a new list. Both accept key= and reverse=.
Two Pointer: Two indices (usually left=0, right=len-1) moving toward each other or in same direction. Avoids nested loops — O(n) instead of O(n²).
Sliding Window: Fixed or variable-size window moving across the array. Track window state as you expand/shrink rather than recalculating each time.
Prefix Sum: Build a running total array. Allows O(1) range sum queries instead of O(n) each time.
Kadane's Algorithm: Max subarray sum in O(n). At each index, decide: extend current subarray or start fresh from current element.
// Practice Problems
P-01Easy
You're building a feature for a food delivery app. Users can filter restaurants by rating. Given a list of restaurant ratings and a minimum rating threshold, return a new list with only the qualifying restaurants' ratings, sorted highest to lowest.
▾

Input: ratings = [4.2, 3.8, 4.7, 2.9, 4.5, 3.1], min_rating = 4.0

Output: [4.7, 4.5, 4.2]

Do not modify the original list. The output should be a new list.

Constraints: 1 ≤ len(ratings) ≤ 10⁴ | 0.0 ≤ rating ≤ 5.0 | 0.0 ≤ min_rating ≤ 5.0
→ Approach hint
Think about what Python tool lets you filter an iterable based on a condition in one line. Then apply sorting with a direction. Key question: what does sorted() return vs .sort()? Which one preserves the original?
P-02Medium
A stock trading platform wants to find the maximum profit possible from a single buy-sell transaction. Given daily prices for a stock over N days, find the maximum profit. You must buy before you sell.
▾

Input: prices = [120, 80, 100, 95, 140, 60, 150]

Output: 90 (buy at 60, sell at 150)

If no profit is possible, return 0. You cannot sell before buying.

Constraints: 1 ≤ len(prices) ≤ 10⁵ | 1 ≤ prices[i] ≤ 10⁴
→ Approach hint
Brute force is O(n²) — check every pair. Can you do better? Think about what you need to track as you move left to right: the minimum price seen so far. At each step, what's the profit if you sell today? Key insight: you don't need to look back if you already know the cheapest buy price up to this point.
P-03Medium
A logistics company tracks daily delivery counts. They want to know the maximum total deliveries achievable in any 7-consecutive-day window across a 30-day period, to identify their peak week.
▾

Input: deliveries = [12, 8, 15, 20, 7, 18, 25, 10, 14, 19, ...] (30 values)

Output: Maximum sum of any 7 consecutive days

The window must be exactly 7 days wide. Return the maximum sum found.

Constraints: len(deliveries) = 30 | 0 ≤ deliveries[i] ≤ 1000
→ Approach hint
If you recalculate the sum from scratch for every window, that's O(n*k). Sliding window idea: when you move one step forward, what changes in your sum? You lose the leftmost element and gain a new rightmost element. Can you update the sum in O(1) instead of recalculating? Start with the first window's sum, then slide.
// Interview tip — Lists
When asked to find pairs, triplets, or subarrays — your first instinct should be can two pointers solve this? Sorting first often unlocks two-pointer solutions. Also: interviewers notice when you use sorted(arr, key=lambda x: x[1]) cleanly — it shows Python fluency, not just algorithm knowledge.
02

Strings

Immutability: Strings cannot be changed in place. s[0] = 'x' throws an error. Concatenation creates a new string. Use list(s) → modify → ''.join() for in-place-style operations.
Common methods: split(), join(), strip(), replace(), startswith(), endswith(), lower(), upper() — all return new strings.
ord() & chr(): ord('a') → 97. chr(97) → 'a'. Used when you need to work with character positions in the alphabet — e.g., ord(c) - ord('a') gives 0-based index.
Slicing & reversal: s[::-1] reverses a string. s[i:j] gives substring from i to j-1. Strings support all sequence operations.
Palindrome check: s == s[::-1] is the Pythonic way. For two-pointer: compare characters from both ends moving inward.
Anagram check: Two strings are anagrams if they have the same character frequencies. sorted(s1) == sorted(s2) works but O(n log n). Better: use Counter.
// Practice Problems
P-04Easy
A username validation system needs to check if any username is a "mirror" — meaning it reads the same forwards and backwards, ignoring spaces and case. Given a username string, determine if it qualifies.
▾

Input: "Race Car"

Output: True

Ignore spaces and case. Only consider alphanumeric characters. "A man a plan a canal Panama" → True.

Constraints: 1 ≤ len(s) ≤ 10⁵ | May contain spaces, punctuation, mixed case
→ Approach hint
Before checking, clean the string. Think in steps: (1) remove everything that isn't alphanumeric — what Python string method or list comprehension helps here? (2) normalize case. (3) check palindrome. Bonus: can you do this in one clean line using slicing?
P-05Medium
A plagiarism detection tool checks if two submitted essays are anagrams — same characters used, different order. Given two strings (lowercased, no punctuation), determine if one is a rearrangement of the other.
▾

Input: s1 = "listen", s2 = "silent"

Output: True

s1 = "hello", s2 = "world" → False. Only lowercase a-z, no spaces.

Constraints: 1 ≤ len(s1), len(s2) ≤ 5×10⁴ | Only lowercase a-z
→ Approach hint
Three approaches — think about the tradeoff: (1) Sort both and compare — O(n log n). (2) Frequency dict manually — O(n). (3) Counter — cleanest, O(n). First check: if lengths differ, what's the answer immediately?
P-06Hard
A search engine needs to find the shortest substring in a document that contains all required keywords. Given a large document string and a list of keywords, find the minimum length substring containing all keywords at least once.
▾

Input: s = "ADOBECODEBANC", keywords = ["A", "B", "C"]

Output: "BANC"

If no valid substring exists, return an empty string.

Constraints: 1 ≤ len(s) ≤ 10⁵ | 1 ≤ len(keywords) ≤ 100
→ Approach hint
Variable sliding window. Expand right until all keywords are covered. Then shrink from the left while still covering all keywords — that's your candidate. Record minimum and repeat. Key question: what data structure tracks how many of each keyword you currently have vs how many you need?
// Interview tip — Strings
Always clarify: ASCII or Unicode? Case-sensitive? Special characters? Never concatenate strings in a loop — use a list and ''.join(list) at the end. String concatenation in a loop is O(n²) due to immutability.
03

Dictionaries & Sets

Core dict ops: .get(key, default) avoids KeyError. .items() for key-value pairs. .setdefault(key, []) initializes if missing. Dict lookup is O(1) average.
Frequency counting: The most common dict pattern — count occurrences of elements. Used in anagram, most frequent element, and character problems constantly.
Set for O(1) lookup: set() gives O(1) membership check. Operations: & (intersection), | (union), - (difference).
Dict as cache/memo: Store computed results to avoid recomputation. Key pattern for dynamic programming and avoiding repeated work.
// Practice Problems
P-07Easy
An e-commerce platform logs every product viewed by a user. Given a list of product prices and a target budget code, find the two products that together sum to the target. Return their indices.
▾

Input: products = [200, 450, 300, 150, 500], target = 600

Output: [0, 2] (200 + 300 = 600)

Exactly one valid pair exists. You may not use the same element twice.

Constraints: 2 ≤ len(products) ≤ 10⁴ | Values and target are positive integers
→ Approach hint
Brute force: every pair — O(n²). One-pass approach: at each element you know what you need (target - current). If you could check in O(1) whether that value exists and where — what data structure gives you that as you iterate?
P-08Medium
A content moderation system processes a stream of words from user comments. After processing, return the top 3 most frequently used words, excluding a given set of stopwords like "the", "a", "is".
▾

Input: words = ["the", "sky", "is", "blue", "the", "sky", "is", "clear", "sky"], stopwords = {"the", "is"}

Output: ["sky", "blue", "clear"]

Sort by frequency descending, then alphabetically for ties.

Constraints: 1 ≤ len(words) ≤ 5×10⁴ | All lowercase | k = 3
→ Approach hint
Step 1: Filter stopwords — stopwords is a set, membership check O(1). Step 2: Count frequencies — which collections tool is built exactly for this? Step 3: Get top-k — that tool has a direct method. Tricky: same frequency ties — how do you sort by frequency descending AND alphabetically ascending simultaneously?
P-09Medium
A social network finds mutual connections between users. Given two users' friend lists (with possible duplicates from import errors), return the unique common friends, and friends unique to each user.
▾

Input: user_a = [101, 204, 305, 101, 406], user_b = [204, 305, 507, 204]

Output: {"mutual": [204, 305], "only_a": [101, 406], "only_b": [507]}

Handle duplicate IDs. Output lists can be in any order.

Constraints: 0 ≤ len(list) ≤ 10⁴ | User IDs are positive integers
→ Approach hint
Convert both lists to sets — duplicates disappear automatically. Then set operations: mutual = &, only_a = -, only_b = reverse -. Think further: if the problem required preserving original order, could you still use sets?
// Interview tip — Dicts & Sets
Trigger words: "count / frequency / occurrence" → dict or Counter. "unique / duplicate / seen before / exists" → set. These two pattern triggers solve roughly 40% of easy/medium problems.
04

Collections Module

Counter: Counter(list) counts frequencies. .most_common(k) returns top-k. Supports arithmetic: c1 + c2, c1 - c2.
defaultdict: Auto-initializes missing keys — defaultdict(list), defaultdict(int), defaultdict(set). Eliminates if key not in d boilerplate.
deque: O(1) append/pop from both ends. Unlike list where pop(0) is O(n). Use for sliding window, BFS queues, fixed-size buffers. deque(maxlen=k) auto-evicts oldest.
// Practice Problems
P-10Easy
A news app tracks which articles each user reads. Given a flat log of (user_id, article_id) pairs, group all article IDs read by each user. Return a dict where each key is a user and value is their list of articles.
▾

Input: logs = [(1, "a1"), (2, "a3"), (1, "a2"), (3, "a1"), (2, "a1")]

Output: {1: ["a1", "a2"], 2: ["a3", "a1"], 3: ["a1"]}

Order of articles per user should match order of appearance in logs.

Constraints: 1 ≤ len(logs) ≤ 10⁵
→ Approach hint
defaultdict(list) auto-creates an empty list when a new key is first accessed — no if user not in d needed. Iterate logs, unpack tuple, append directly. Think: how would this change if you needed unique articles per user?
P-11Medium
A trading terminal displays a rolling average of the last K stock prices. As new prices stream in one by one, the window slides forward. Given a stream of prices and window size K, return the list of averages for each window position.
▾

Input: prices = [100, 110, 105, 120, 95, 130], k = 3

Output: [105.0, 111.67, 106.67, 115.0]

Return results rounded to 2 decimal places.

Constraints: 1 ≤ k ≤ len(prices) ≤ 10⁵
→ Approach hint
deque(maxlen=k) — when you append to a full maxlen deque, it auto-drops the oldest element. Append new price → if len(deque) == k, calculate average. No manual index tracking. Alternative: can you solve with just a running sum variable?
// Interview tip — Collections
Use defaultdict when you're always initializing missing keys the same way. Use deque for O(1) on both ends. Use Counter when counting is the core problem. Don't reach for them when a plain dict works fine — overusing collections looks like pattern-matching, not understanding.
05

Python Built-ins

enumerate(): Index + value together. for i, val in enumerate(arr) — no more range(len(arr)). Optional start: enumerate(arr, start=1).
zip(): Pairs elements from multiple iterables. Stops at shortest. zip(*matrix) transposes a 2D list. dict(zip(keys, values)) builds a dict from two lists.
map() & filter(): map(func, iterable) applies function to each element. filter(func, iterable) keeps truthy results. Both return iterators — wrap in list().
any() & all(): any() — at least one truthy. all() — all truthy. Both short-circuit. Commonly used with generator expressions for clean one-liners.
// Practice Problems
P-12Easy
An HR system has two equal-length lists — employee names and their departments. Build a lookup dictionary from these. Then, given a list of names to query, return their departments or "Not Found" if missing.
▾

Input: names = ["Amit", "Priya", "Ravi"], depts = ["Engineering", "Design", "Marketing"], query = ["Priya", "Kiran"]

Output: ["Design", "Not Found"]

Constraints: len(names) == len(depts) | 1 ≤ len(query) ≤ 10³
→ Approach hint
zip(names, depts) pairs them — then dict() wraps it. One line. Query with .get(name, "Not Found"). Challenge: can you write the entire solution in 2 lines?
P-13Medium
A QA system checks server health. Given a list of server status dicts (id, cpu, memory, active), determine if ALL active servers are within thresholds (cpu < 80, memory < 75). Return True/False and list any failing server IDs.
▾

Input: [{"id":"s1","cpu":45,"memory":60,"active":True}, {"id":"s2","cpu":85,"memory":70,"active":True}, {"id":"s3","cpu":90,"memory":80,"active":False}]

Output: {"healthy": False, "failing": ["s2"]}

Inactive servers (active: False) must be ignored completely.

Constraints: 1 ≤ len(servers) ≤ 10³ | cpu and memory are 0–100
→ Approach hint
Filter active servers first. Then all(is_healthy(s) for s in active) short-circuits at first failure. Build failing list with a comprehension. Think: what's the difference between all() and a manual loop with a flag?
// Interview tip — Built-ins
Using built-ins signals Python fluency. But don't force them — a clear for-loop beats a convoluted one-liner with nested map(filter(lambda...)). Rule: if you need to explain what your one-liner does, it's too clever.
